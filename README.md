# WorkQueue - Multi-Tenant Task Management System

A full-stack task management application with strict Role-Based Access Control (RBAC) and Tenant Isolation. Built with ASP.NET Core (Web API) and Angular.

## Features
* **Tenant Isolation:** Users can only access data (tasks, users, comments) belonging to their specific Organization. Enforced globally via EF Core Query Filters.
* **Role-Based Access Control:** * `Manager`: Can create tasks, edit all task details, and assign users.
  * `Member`: Can only view tasks and change the status of tasks assigned specifically to them.
* **Real-time Task Discussion:** Built-in commenting system for each work item.
* **JWT Authentication:** Secure stateless authentication.

## Tech Stack
* **Backend:** C#, ASP.NET Core 10 Web API, Entity Framework Core, SQL Server.
* **Frontend:** Angular 17, TypeScript, RxJS.
* **Testing:** xUnit, Moq, EF Core In-Memory Database.

## Getting Started

### Prerequisites
* [.NET 10](https://dotnet.microsoft.com/download)
* [Node.js](https://nodejs.org/) (v18 or higher)
* [Angular CLI](https://angular.io/cli) (`npm install -g @angular/cli`)
* SQL Server (LocalDB or Docker)

### 1. Database Setup & Seeding
The application uses EF Core Code-First migrations. The database will automatically be created and seeded with test data upon the first run.

Test Accounts generated by default (Password for all: `dev_password`):
* `manager.a@test.com` (Organization A - Manager)
* `member.a@test.com` (Organization A - Member)
* `manager.b@test.com` (Organization B - Manager)

### 2. Backend Configuration (No Secrets Committed)
Navigate to the backend project folder:
```bash
cd WorkQueue
Initialize user secrets for your local environment to set up the JWT Key safely:

Bash
dotnet user-secrets init
dotnet user-secrets set "Jwt:Key" "your_super_secret_key_minimum_16_characters"
Run the API:

Bash
dotnet run
The API will start at https://localhost:7122. You can explore the endpoints via Swagger at https://localhost:7122/swagger.

3. Frontend Setup
Open a new terminal and navigate to the frontend directory:

Bash
cd workqueue-ui
npm install
npm start
[ng serve -o]
The application will open in your browser at http://localhost:4200.

🧪 Running Tests
The project includes xUnit tests ensuring core business logic and tenant isolation work correctly.

Bash
dotnet test
